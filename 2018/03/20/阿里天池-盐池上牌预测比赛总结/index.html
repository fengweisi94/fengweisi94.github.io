阿里天池-盐池上牌预测比赛总结

发表于 2018-03-20

1.比赛大意

选手利用历史某3年的汽车日上牌数据，预测某2年每天的汽车上牌数。初赛将挑选出5个汽车品牌，给出这些品牌每天的上牌数，当天是星期几，来预测5个汽车品牌未来每天的上牌总数。复赛将挑选出10个汽车品牌，仍旧给出品牌每天的上牌数，当天是星期几，来预测10个汽车品牌未来每天的上牌数。

2.详细做法

  1. 特征提取

    • 将脱敏日期反推回真实日期（通过节假日放假情况推断），并将日期转换为三个字段年、月、日
    • 增加一个字段用来记录日子类型，比如0：工作日，1：休息日，2：节假日
    • 从数据可以看出一般节假日过后的第一天上牌的数量会猛增，认为是连续的放假使上牌量累计，所以增加一个字段来表示当天与上一个工作日隔了几天。
    • 天气，认为天气会影响人买车从而影响上牌（复试要求不得使用外部数据）

  2. 模型建立

    • ARIMA：效果不佳+不怎么会使用
    • prophet：facebook的时间序列模型，效果一般，使用简单，可视化强，可以很清楚的看到年关系，月关系，星期关系
    • GBM,XGBOOST,LIGHTGBM：最终比赛决定使用这三个监督模型

  3. 模型融合

    • 平均融合，加权融合：根据模型好坏融合
    • stacking：初次使用，没有经验，失误了，错误率大增，遗憾

2.GBM
GBM的参数大致可以分为三类：

  1. 树特有的参数：
    • min_samples_split:表示一个结点分割需要的最小样本数
    • min_samples_leaf:表示叶子结点需要的最小样本数
    • min_weight_fraction_leaf:等价于min_samples_split，它是总体样本的百分比
    • max_depth:树的最大高度
    • max_leaf_nodes:一颗树中能包含的最大叶子结点个数
    • max_features:做特征选择时所能考虑的最多特征数
  2. 提升操作参数：
    • learning_rate:学习率
    • n_estimators:数的个数
    • subsample:给每棵树多少的样本，百分比
  3. 其他参数：
    • loss:损失函数
    • init:
    • random_state:随机种子，模型复现
    • verbose:模型拟合后要打印输出的类型
    • warm_start:热启动
    • presort:选择是否预分类去更快分割结点

调参过程：

  1. 先选择一个较高的学习率learning_rate，通常选择0.1,对于不同问题一般是在0.05-0.2
  2. 确定在当前学习率下最优的树的个数n_estimators，通常是40-70。如果最优值在20左右，那就需要尝试更小的学习率，如果最优值比100高，那就需要尝试更高的学习率
  3. 调整树特有的参数
    参数初始化：min_samples_split = 500, min_sample_leaf=50, max_depth=8, max_features=’sqrt’, subsample=0.8
    1. 调整树最大高度max_depth(通常是5-8)和最小样本数分割num_samples_split(通常是总样本数的0.5%-1%)
    2. 调整叶子结点所要包含的最小样数min_samples_leaf(30-100)
    3. 调整分割最大特征数max_feature
    4. 调整subsample(0.5-0.9)
  4. 降低学习率learning_rate，成比例地提高树的个数n_estimators

示例如下：

  1. 先确定learning_rate为0.1
  2. 在n_estimators:range(20,81,10)中搜寻最优值,比如60
  3. ‘max_depth’:range(5,16,2), ‘min_samples_split’:range(200,1001,200)
  4. ‘min_samples_leaf’:range(30,71,10)
  5. ‘max_features’:range(7,20,2)
  6. ‘subsample’:[0.6,0.7,0.75,0.8,0.85,0.9]
  7. 调整learning_rate和n_estimators,固定其他值，比如调整learning_rate为0.05，则n_estimators成比例增加为120。

上述参数选择过程可以用GridSearchCV函数实现

3.XGBOOST
XGBOOST的参数大致可以分为三类：

  1. 普通参数：

    • booster[default=gbtree]:基模型,分为gbtree和gblinear，通常gbtree效果更好
    • silent[default=0]:1表示不输出信息
    • nthread[default to maximun number of threads available if not set]:用于并行处理的线程数

  2. 提升参数：

    • eta[default=0.3]:类似GBM的learning_rate，通常是0.01-0.2
    • min_child_weight[default=1]:类似min_child_leaf，叶子结点最少需要多少样本权重和。（个人理解：因为XGBOOST可以处理缺失值，所以联系决策树对缺失值的处理，就能想到为什么这里是权重和）
    • max_depth[default=6]:树最大高度，通常是3-10
    • max_leaf_nodes:一颗树中能包含的最大叶子结点个数
    • gamma[default=0]:分割后loss减小>gamma才分割
    • max_delta_step[default=0]:不清楚，通常用不到
    • subsample[default=1]:给每棵树多少的样本，百分比
    • colsample_bytree[default=1]:类似于GBM的max_features。给每棵树多少的特征
    • colsample_bylevel[default=1]:每次分割用多少的特征
    • lambda[default=1]:L2正则化
    • alpha[default=0]:L1正则化
    • scale_pos_weight[default=1]:帮助类别不均衡收敛

  3. 学习任务参数

    • objective[default=reg:linear]:损失函数-binary:logitic,multi:softmax,multi:softprob
    • eval_metric[default according to objective]:用于验证集的度量标准-rmse,mae,logloss,error,merror,mlogloss,auc
    • seed:随机种子，模型复现

sklearn对XGBOOST做了包装XGBClassifier，加入了n_estimators。相关参数做了改变：

  • eta->learning_rate
  • lambda->reg_lambda
  • alpha->reg_alpha

调参过程：

  1. 选择一个较高的学习率,通常是0.1,区间是0.05-0.3。确定当前学习率下最优树的个数
  2. 调整树特有的参数(初始化：max_depth=5,min_child_weight=1,gamma=0,subsample=0.8,colsample_bytree=0.8,)
    • 调整max_depth(3-10)和min_child_weight(1-15)
    • 调整gamma(视情况而定0-1)
    • 调整subsample、colsample_bytree(0.5-0.9)
  3. 调整正则化参数lambda,alpha(0,0.0001,0.001,0.01,0.05,1,10)
  4. 降低学习率learning_rate，成比例地提高树的个数n_estimators

4.stack


对于每一轮的5-fold,Model1都要做满5次的训练和预测。
假设Train Data有1000行（对应图中的上层部分）。
每一次的fold，都会产生800行的小train,200行的小test。我们用Model1来训练800行的小train,然后预测200行小test。预测的结果是长度为200的预测值。
这样的动作走5次！ 长度为200 的预测值 X 5 = 1000 预测值，刚好和Train data长度吻合。这个1000预测值是Model1产生的，我们先存着，因为，一会让它将是第二层模型的训练来源。
重点：这一步产生的预测值我们可以转成 1000 X 1 （1000 行，1列），记作 P1 (大写P)

接着说Test Data有400行。(对应图中的下层部分）
每1次的fold，800行小train训练出来的Model1要去预测我们全部的Test Data（全部！因为Test Data没有加入5-fold，所以每次都是全部！）。此时，Model 1的预测结果是长度为400的预测值。
这样的动作走5次！我们可以得到一个 5 X 400 的预测值矩阵。然后我们根据行来就平均值，最后得到一个1 X 400 的平均预测值。
重点：这一步产生的预测值我们可以转成 400 X 1 （400行，1列），记作 p1 (小写p)

对第一层其他模型也做相同操作。如果总共有3个模型，则Train Data的预测矩阵为1000 X 3(P1,P2,P3)和Test Data预测值矩阵400 X 3(p1,p2,p3)。

到第二层：
来自Train Data的预测值矩阵 1000 X 3 作为你的Train Data，训练第二层的模型
来自Test Data预测值矩阵 400 X 3 就是你的Test Data，用训练好的模型来预测他们。
